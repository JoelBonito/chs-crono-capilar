# Documentacao de Planejamento

## Objetivo
Garantir execução consistente do projeto, com rastreabilidade total entre planejamento, backlog e sprints.

## Ordem de leitura recomendada
1. `01-product-brief.md`
2. `02-prd.md`
3. `03-ux-concept.md`
4. `03.5-visual-mockups.md`
5. `04-architecture.md`
6. `05-security.md`
7. `06-stack.md`
8. `07-design-system.md`
9. `08-backlog.md`
10. `09-sprint-planning.md`

## Fonte de verdade de execução
- `08-backlog.md` é a referência oficial de status de desenvolvimento.
- `09-sprint-planning.md` define a sequência operacional dos sprints.

## Regras de auditoria
1. Nenhuma task de sprint sem story correspondente no backlog.
2. Toda story concluída deve conter evidência no backlog.
3. Fechamento de sprint exige sincronização completa com backlog.
4. Gaps identificados devem estar mapeados para stories de mitigação.

## Critério para iniciar caderno de testes
Iniciar somente quando 100% das stories estiverem em `Done` no `08-backlog.md`.
